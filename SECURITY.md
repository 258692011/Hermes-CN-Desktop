# Security Policy

## Supported Versions

当前项目仍处于 alpha 阶段。安全修复优先进入 `main`，并在需要时随下一个预发布或补丁版本发布。

## Reporting a Vulnerability

请不要把未公开的安全漏洞直接发到公开 Issue。你可以通过 GitHub 的私有漏洞报告能力提交，或在项目主页公布的维护者联系方式中选择私下渠道联系维护者。

报告时请尽量包含：

- 受影响版本、平台和安装方式。
- 可复现步骤或最小 PoC。
- 影响范围，例如本地文件读取、任意命令执行、凭证泄漏、SSRF、路径穿越或更新链路绕过。
- 你是否已经公开披露或计划披露。

维护者会尽快确认问题、评估影响，并在修复可用后协调披露。
